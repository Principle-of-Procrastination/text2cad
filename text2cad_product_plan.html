03 · Decisions

不可轻易回退的决策

LLM 不执行代码

模型只产生白名单语义 capability。禁止输出并执行 C#、VBA、宏或原始 SOLIDWORKS API。

计划绑定文档 revision

用户确认不可变 plan hash。预览后文档或选择变化时，旧计划立即失效。

原生特征才算完成

结果必须可 Edit Sketch、Edit Feature、Rebuild、保存并重开；Imported Body 不通过验收。

04 · Safety loop

安全执行闭环

每一层都可拒绝请求；确认不是一颗跳过校验的“执行”按钮。

1

Snapshot

读取文档、配置、单位、特征图和选择稳定引用。

2

Plan

Host 生成版本化、显式单位的语义计划。

3

Validate

Add-in 复核 schema、capability、范围和 revision。

4

Preview

展示新增/修改、before/after、依赖与风险。

5

Execute

确认 plan hash 后，在 Undo Object 中串行执行。

6

Verify

Rebuild 与后置条件全过才成功，否则恢复并诊断。

06 · MVP gates

MVP 必须通过

✓

宿主隔离

Host 崩溃或断线时，SOLIDWORKS 仍稳定可用。

✓

非法计划无法触达 COM

未知 operation、非法单位、越界数值和跨文档引用在执行前被拒绝。

✓

确认具有一致性

用户确认 plan hash；document revision 改变会阻止旧计划执行。

✓

结果原生且可编辑

保存并重开后，草图、尺寸和特征仍能编辑并成功重建。

✓

失败可诊断

任何步骤失败都有操作级证据、结构化错误和 correlation ID。

✓

恢复可验证

失败后证明已恢复执行前状态；否则明确要求人工检查。

07 · Risks

核心风险

SOLIDWORKS API 与拓扑脆弱性

用持久引用、类型/拓扑预期、真机 golden models 和版本矩阵控制；不依赖显示名称。

把演示误当产品

板和孔只是工程 fixture；首个市场工作流必须经用户访谈、频率、风险和付费意愿验证。

LLM 幻觉越过安全边界

模型无执行权；双层 validator、capability registry、显式单位和模糊测试构成硬边界。

过早扩展范围

在 Part 的窄 capability 闭环稳定前，不承诺通用装配体、工程图、多 CAD 或全自动拓扑推理。
